## What Was Done

### Problem Solved

The Fraunhofer IIS HPC cluster (CentOS 7, glibc 2.17) cannot natively run modern ML packages:
- PyTorch 2.7.1, JAX 0.5.3, polars, orbax-checkpoint all require glibc >= 2.28
- Previous attempts with `--no-deps` or version downgrades caused cascading dependency failures

### Solution: conda `sysroot_linux-64=2.28` + `patchelf`

The definitive approach:
1. Install micromamba (lightweight conda)
2. Create conda env with Python 3.11 + `sysroot_linux-64=2.28` + `patchelf` + `libstdcxx-ng`
3. Patchelf Python binary to use sysroot's glibc 2.28 dynamic linker
4. Now pip sees glibc 2.28 and installs ALL wheels normally (no hacks!)
5. Generate `activate_hpc.sh` for SLURM jobs to source at runtime

## How activate_hpc.sh Works

This file is auto-generated by `setup_hpc_env.sh` and sets:

```bash
export PATH="${VENV}/bin:$PATH"
export LD_LIBRARY_PATH="${SYSROOT}/lib64:${SYSROOT}/usr/lib64:${VENV}/lib:$LD_LIBRARY_PATH"
export CONDA_PREFIX="${VENV}"
export HF_HOME="/data/beegfs/home/saifi/.cache/huggingface"
export XLA_PYTHON_CLIENT_MEM_FRACTION=0.90
export XLA_PYTHON_CLIENT_PREALLOCATE=true
export WANDB_PROJECT="rlt-ur5e"
```

## Known Edge Cases

### rerun-sdk (lerobot dependency)

`lerobot` optionally depends on `rerun-sdk` (3D visualization) which needs X11/OpenGL.
The setup handles this automatically:
- Tries full install first
- If rerun-sdk fails, installs lerobot with `--no-deps` but manually adds ALL runtime deps
- This is the ONLY place `--no-deps` is used (tightly controlled)

### GPU driver version

If compute nodes have NVIDIA driver < 525, change the PyTorch index from `cu124` to `cu118`
in `setup_hpc_env.sh` (lines with `download.pytorch.org/whl/cu124`).

## Troubleshooting Quick Reference

| Issue | Fix |
|-------|-----|
| GLIBC_2.28 not found | `source activate_hpc.sh` before running Python |
| No module named X | `pip install X` (glibc 2.28 active, wheels will work) |
| GPU not found | `srun --gres=gpu:1 nvidia-smi` (need driver >= 525) |
| Rebuild from scratch | `rm -rf .venv && bash setup_hpc_env.sh` |
| W&B offline | `source activate_hpc.sh && wandb login` |
| Check glibc version | `python -c "import ctypes; libc=ctypes.CDLL('libc.so.6'); f=libc.gnu_get_libc_version; f.restype=ctypes.c_char_p; print(f().decode())"` |
